1. 게스트(db)에 정수형 값 추가
예를들어 게스트 클라이언트 계정이 4개가 있고 각각 1, 1000 , 2000, 3000이 초기값으로 
저장되어 있고 주문할 때마다 1씩 증가. 매우 간단함. 새로운 값은 중복되면 안되기 때문에 구현할 때 현재까지 증가된 정수형 값이 다시 게스트 DB에 저장되야 할듯. 장점으로는 기존 데이터베이스 구조에서 크게 변경되지 않을거 같다. 편리할 거같음 단점도 게스트가 엄청 많은게 아니라면 없을거 같음.

2. order_no를 그대로 두고 새로운 no를 order에 추가.
1번과 비슷한 느낌인데 1번이 더 괜찮을 거같은 느낌?. 결국 Order_detail 얘가 더 복잡해질거 같음.

3. 서버에서 주문을 DB에 저장하고 order_no를 받아오고 다시 클라이언트에게 전송.
클라이언트가 서버에게 주문을 요청하고, (Order를 보냄). DB에 저장할때 Order_no를 가져와서 다시 클라이언트에게 Order를 보내고 Order_No를 Order_detail에 저장하여 다시 서버에게 Order_detail을 전송하여 서버는 DB에 저장. 장점으로는 서버에서 직접 Order_no를 생성하고 클라이언트에게 전달하기 때문에 주문번호가 꼬일일이 없음.
단점으로는 클라이언트에게 다시 전송하는 것에서 시간이 좀 소모 될거같고 복잡함.

4. 서버에 주문번호 전역변수를 추가한다. 
주문번호 값을 가져올려면 DB에 요청을해서 값을 가져와야할 거같음. 어차피 다시 주문번호를 client에 전송하는거면 3번처럼 비슷하고 의미가 없지 않나

결론 : 생각해보니 3번 방식으로 해도 구현 문제는 없을거 같음. 서버 측에서 주문번호를 제어하기 때문에 논리적으로 도문제가 없음. 하지만 클라이언트에게 다시 정보를 전송하는 것에서 추가적인 트래픽이나 시간이 소모 될거 같아 통신이 복잡해짐. 아니면 1번 방식이 조금 더 괜찮아 보임. 저 정수값을 Order_no로 하면 그냥 문제 없이 굴러갈거 같음.
